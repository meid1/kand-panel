# Changelog — Kand

Версия панели показывается в шапке и на экране входа. При каждом обновлении:
бампим `APP_VERSION` в `apps/api/public/app.js` и добавляем строку сюда.

## v0.21.0 — 2026-07-04
- **🎁 Подарочные подписки:** выпуск кодов пачкой, клиент активирует в боте (тем же полем,
  что и промокод), идемпотентное погашение. Вкладка «Подарки».
- **☑️ Bulk-операции:** массовый выбор клиентов галочками + продлить/баланс/бан/разбан/в промо-группу.
- **🎯 Промо-группы:** сегменты клиентов со скидкой (%), скидка сама применяется при оплате и
  продлении (в т.ч. с баланса и звёздами); назначение через массовый выбор.
- **📢 UTM-кампании:** ссылка `/c/<код>` считает клик и ведёт в бота; воронка клик→регистрация→оплата→доход.
- **⭐ Telegram Stars:** нативная оплата звёздами (валюта XTR) прямо в Telegram, без внешней платёжки;
  курс ₽/звезда настраивается. Кнопка появляется в меню оплаты, если включено.
- **💳 Рекуррент/сохранённые карты (YooKassa):** автосписание при автопродлении без участия клиента
  (по умолчанию выключено; карта сохраняется только при включённом рекурренте).

## v0.20.0 — 2026-07-04
- **Надёжность:** атомарное списание баланса (без гонок двойного списания) + компенсация
  при сбое выдачи; атомарный реф-бонус; HTTP-таймауты (Telegram/платёжки); мультибот —
  проверка принадлежности тарифа тенанту, авто-рестарт упавшего бота, изоляция FSM по юзеру;
  бот — блок-гейт подписки и guard нетекстовых сообщений; админка — устойчивый api() и дедуп синка.
- **Веб-конфигуратор установки** (`/setup.html`): выбор домена/протоколов/возможностей → готовая
  команда. `install.sh` понимает `--protocols/--disable/--enable`; FeaturesService гейтит вкладки.
- **Мастер настройки** (онбординг) на дашборде: 5 шагов с прогрессом (`GET /onboarding`).
- **Поддержка (тикеты):** модель Ticket/TicketMessage, вкладка «Поддержка» (список/тред/ответ),
  флоу в боте (клиент пишет → тикет), доставка ответа клиенту в Telegram. Гейтится флагом `tickets`.
- **Живое демо** demo.kandpanel.com (изолированный контейнер, синтетика, авто-сброс).
- Модель GiftCode (подарочные подписки) — задел под фичу.

## v0.19.0 — 2026-07-04
- Синхронизация теперь в РЕАЛЬНОМ ВРЕМЕНИ, а не раз в 3 мин: свежие оплаты/продления
  из базы бота подтягиваются при каждом открытии дашборда и списка клиентов (лёгкий
  догон по курсору), а карточка клиента синкается точечно при открытии — срок/бан/
  платежи всегда актуальны в момент просмотра. Фоновый проход оставлен как страховка
  (регистрации без оплаты, ручные баны). Эндпоинты POST /sync/light, /sync/user/:tgId,
  /sync/user-by-id/:id.

## v0.18.1 — 2026-07-04
- Фикс воронки: «Получили доступ» считалось только по устройствам в самой Kand
  (в гибриде ключи живут во внешнем cdn_api → выходило меньше, чем «Оплатили»).
  Теперь = есть ключ ИЛИ заведена подписка (expireAt) → доступ ≥ оплатили, как и должно.

## v0.18.0 — 2026-07-04
- Живая синхронизация из БД бота: новый модуль `sync` тянет из внешней БД (vpn_db)
  новых клиентов, продления (срок подписки), баны и новые платежи в панель каждые
  3 мин (и по кнопке «Обновить» на дашборде). Раньше данные были разовым снимком и
  устаревали. Инкрементально по курсору (Setting), дедуп платежей по invoiceId,
  строго одностороннее чтение. Гейт: BRIDGE_DB_* + SYNC_VPNDB_TENANT_ID.
- На дашборде — индикатор свежести «🔄 данные: N мин назад» + ручной запуск.

## v0.15.0 — 2026-07-03

  Фикс: порт брался из `port`, а бэкенд отдаёт `grpc_port` — теперь показывается верно.
- Иконки навигации (v0.14): emoji заменены на аккуратные линейные SVG-иконки как у эталона;
  тумблер показа сервера в подписке, переименование и порядок ↑↓ живых нод.

## v0.10.0 — 2026-07-03
- Дизайн под эталон: sidebar-навигация с иконками, кнопки/поля/карточки/таблицы/пилюли —
  ховеры, фокус, аккуратные акценты (#22D3EE / Inter). Карточка клиента — модальное окно.
- Дашборд: графики выручки и регистраций с ОСЯМИ (сетка, подписи X=дата / Y=значение), тултипы.
  Метрики: ARPPU за 30 дн (с платящего), LTV (с платящего за всё), отток только среди платящих
  — с расшифровками (ⓘ). Воронка зашли→доступ→оплатили. Реальные даты регистраций (бэкфилл).
- Серверы: Хост/домен необязателен (=IP) с подсказкой; ручное добавление готового сервера
  (свои pbk/sid/sni/порты); порядок ↑↓ (очередь в подписке); имя ноды в подписке; показ живых
  серверов внешнего бэкенда.
- Вкладки «Трафик» (топ по трафику) и «HWID» (клиенты с макс. устройств).
- Мобильный адаптив, no-cache для html/js, скрытие вкладок (ui.hidden_tabs).

## v0.9.0 — 2026-07-03
- Дашборд «здоровье бизнеса»: выручка/мес с ростом к пред. месяцу, активные, отток/мес,
  ARPU, LTV; воронка (зашли → получили доступ → оплатили); графики выручки (линия) и
  регистраций (столбики) за 30 дней; карточка живого бэкенда (онлайн/трафик).
- Раздел «Финансы»: доход по способам/месяцам, учёт расходов/доходов, чистая прибыль.
- Раздел «Платежи»: список с поиском/фильтром по статусу и сменой статуса.
- Карточка клиента: блок/разблок, продление дней, обход +/−, сброс счётчика, тумблер обхода,
  force-enable, выдать/удалить ключ, устройства (копия ссылки/удаление), диагностика.
- Поиск + пагинация клиентов (для тысяч юзеров).
- Управление серверами: проверка (health+пинг), редактирование ноды.
- Промокоды: создание/редактирование/удаление.

  систему. Включается конфигом (`BRIDGE_API_*`, `BRIDGE_DB_*`), иначе no-op.
- Мультибот франшиз, автопродление с баланса, WARP, ручное создание ключа.
- Импорт: сохранение subToken (миграция без смены ссылок подписки).
- UI: тёмно-циановая тема (#22D3EE, Inter), адаптив для мобильного (лента-меню),
  no-cache для html/js, скрытие вкладок через `ui.hidden_tabs`.
