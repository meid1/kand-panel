# Безопасность Kand

## Сообщить об уязвимости
Не открывай публичный issue. Напиши приватно (email/Telegram владельца репозитория) —
исправим и раскроем ответственно.

## Что уже заложено
- Панель↔нода — mTLS (клиент без сертификата нашей CA не подключится) + Bearer JWT.
- Вход в админку — по паролю (`ADMIN_PASSWORD`) → JWT (HS256), анти-брутфорс 5/мин.
- Rate-limit: подписка 30/мин на IP, установщик 20/мин, весь API 120/мин.
- Проверка подписей ВСЕХ платёжных вебхуков (иначе оплата не засчитывается).
- Аудит изменяющих действий (секреты вычищаются).
- Секреты (ключи нод, приватные ключи) не отдаются в API/UI.

## Рекомендации при развёртывании
- Смени `ADMIN_PASSWORD` и `JWT_SECRET` (длинная случайная строка).
- Не коммить `.env`. Панель — только по HTTPS.
- Ноды под нагрузкой — хостинг с анти-DDoS (локальный фаервол не спасает от объёмного флуда).
