# Установка Kand «с нуля» (для новичка)

Пошагово, без опыта. Нужно ~20 минут и сервер (VPS).

## Что понадобится
- **Сервер (VPS)** под панель: Ubuntu 22.04+, 1 CPU / 1 ГБ ОЗУ хватит. (Панель и ноды — это РАЗНЫЕ серверы: панель управляет, ноды раздают VPN.)
- **Домен** для панели (напр. `panel.мойвпн.рф`), направленный A-записью на IP сервера панели.
- Телеграм-бот (токен от @BotFather) — если нужен клиентский бот.

---

## Шаг 1. Подключиться к серверу
```bash
ssh root@IP_СЕРВЕРА
```

## Шаг 2. Поставить Docker (одной командой)
```bash
curl -fsSL https://get.docker.com | sh
```

## Шаг 3. Скачать Kand
```bash
git clone https://github.com/meid1/kand-panel.git /opt/kand
cd /opt/kand
```

## Шаг 4. Настроить `.env`
```bash
cp .env.example .env
nano .env
```
Заполни (остальное можно оставить):
- `PANEL_URL=https://panel.твойдомен` — адрес панели (с https, без слэша в конце);
- `ADMIN_PASSWORD=` — придумай надёжный пароль для входа;
- `JWT_SECRET=` — длинная случайная строка (сгенерируй: `openssl rand -hex 32`);
- `POSTGRES_PASSWORD=` — пароль базы (тоже случайный);
- `BOT_TOKEN=` — токен бота от @BotFather (если нужен клиентский бот).
Сохранить в nano: `Ctrl+O`, `Enter`, `Ctrl+X`.

## Шаг 5. Запустить панель
```bash
docker compose -f infra/docker-compose.prod.yml up -d --build
```
Первый запуск соберёт образ (несколько минут). Проверить, что работает:
```bash
curl -s http://localhost:3000/api/health
```
Должно вернуть `{"ok":true,...}`.

## Шаг 6. Открыть панель по HTTPS
Поставь перед панелью веб-сервер с бесплатным сертификатом. Проще всего Caddy:
```bash
apt install -y caddy
echo 'panel.твойдомен {
    reverse_proxy localhost:3000
}' > /etc/caddy/Caddyfile
systemctl restart caddy
```
Готово — открой `https://panel.твойдомен`, войди по `ADMIN_PASSWORD`.

## Шаг 7. Добавить первый сервер (ноду)
1. Возьми **второй** сервер (в стране, откуда нужен VPN).
2. В панели: **Ноды → добавить** — заполни название, домен/IP, протоколы, роль → «Создать».
3. Панель покажет команду вида `curl -fsSL https://panel.твойдомен/install-node.sh | ... bash`.
4. Выполни эту команду **на сервере ноды** (по SSH). Через минуту нода подключится сама.

## Шаг 8. Подключить клиента
- **Через бота:** клиент пишет боту `/start` → «Подключиться» → получает ссылку-подписку → вставляет в приложение (v2rayNG / Streisand / Hiddify).
- Ссылка с умной маршрутизацией (RU напрямую, YouTube без рекламы): `.../sub/<токен>/xray`.

## (Опционально) Перенос клиентов с другой панели
См. [migration.md](migration.md) — есть готовые пресеты для 3x-ui / Marzban / Remnawave.

## Тексты, кнопки, платёжки, бренд
Всё меняется в веб-админке (вкладки «Тексты и кнопки», «Платёжки», «Бренд») — код трогать не нужно.

## Обновление Kand
```bash
cd /opt/kand && git pull && docker compose -f infra/docker-compose.prod.yml up -d --build
```

## Если что-то не работает
- Логи панели: `docker compose -f infra/docker-compose.prod.yml logs -f api`
- Нода не подключилась: на ноде `journalctl -u kand-agent -u xray -n 50`
- Клиенту «Лимит обхода исчерпан» после переноса — сбрось счётчик (см. migration.md).
