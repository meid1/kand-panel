# Умная маршрутизация и обход (Kand)

Три фичи, отличающие Kand от обычных панелей. Все настраиваются из админки/настроек,
без правки кода.

## 1. RU-сервисы напрямую (мимо VPN)
Российские сайты (банки, госуслуги, маркетплейсы, Яндекс) блокируют иностранные IP.
Поэтому в клиентском конфиге они идут **напрямую**, а не через VPN.

- Работает автоматически: клиент импортирует конфиг с маршрутизацией по адресу
  `https://<панель>/sub/<токен>/xray`.
- Правила: `geosite:category-ru`, `geosite:category-gov-ru`, `geoip:ru` → `direct`.
- Обычный список ссылок (`/sub/<токен>`) — без маршрутизации, для простых клиентов.

## 2. YouTube без рекламы
YouTube-реклама таргетируется по стране выхода. Если направить YouTube через
**российский** выход — рекламы (почти) нет.

Как включить:
1. Заведи ноду **физически в РФ**.
2. При добавлении в админке выбери роль **«YouTube-РФ (без рекламы)»** (`role=yt-ru`).
3. Готово: в умном конфиге весь YouTube (`geosite:youtube`, `googlevideo.com`,
   `ytimg.com`, `ggpht.com`) идёт на эту ноду, остальной трафик — через обычный выход.

## 3. Обход белых списков с лимитом ГБ
«Обход» = трафик к заблокированным/иностранным сайтам через VPN. На него можно
поставить лимит ГБ на клиента.

- Лимит **опционален**. По умолчанию — безлимит.
- Включить общий лимит: настройка `bypass.default_cap_gb` (ГБ). Индивидуальный —
  `capManualGb` у клиента.
- Учёт трафика идёт автоматически (агенты нод → панель, раз в 5 мин).
- При исчерпании: ключи клиента снимаются с нод (обход реально стоит), а в подписке
  сервер меняется на «Лимит обхода исчерпан — докупите в <поддержка>».
- Докупка/списание ГБ — в карточке клиента (кнопки «+ докупить» / «− списать»).
  Списание идёт **без уведомления** клиента.
- Период сбрасывается раз в 30 дней автоматически.

## Защита нод от DDoS
- Всегда: защита ядра от SYN-флуда (sysctl).
- Опционально (`HARDEN_FIREWALL=1` при установке ноды): nftables — порт агента
  только с IP панели, per-IP лимит новых подключений (щедрый, `VPN_CONN_RATE`),
  SSH всегда открыт.
- **Важно:** локальные правила спасают от флуда подключениями, но НЕ от объёмного
  DDoS (десятки Гбит/с забьют канал). Для этого нужен хостинг с анти-DDoS или
  скруббинг (DDoS-Guard, Cloudflare Spectrum) — это выбор хостинга под ноду.

## Свои сайты в маршрутизации (кастом-роутинг)
В админке → «Бренд» → «Маршрутизация — свои сайты» можно добавить домены:
- **Напрямую** — идут мимо VPN (например, локальный банк/сервис);
- **Блокировать** — не открываются (реклама/трекеры);
- **Через VPN** — принудительно через сервер (например, если сайт не открывается напрямую).
Применяется в умном конфиге (`/sub/<token>/xray`), приоритетнее стандартных RU-правил.

## Важно про «обход» и CDN-фронтинг
Kand реализует: VPN + умную маршрутизацию (RU-direct/YouTube) + **учёт лимита ГБ на обход** (метринг, докупка,
авто-стоп). Это уже даёт доступ к заблокированным сайтам через VPN.

Чего в панели НЕТ (by design): **CDN-фронтинг / домен-фронтинг** (клиент → сторонний CDN → ваш origin), который
прячет сам факт использования VPN за легитимным CDN. Это отдельная инфраструктура (нужен свой CDN-аккаунт и origin),
она не переносится в опенсорс-панель. Если нужен такой обход — это ваша частная надстройка поверх нод.
