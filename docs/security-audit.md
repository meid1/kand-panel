# Аудит безопасности Kand

Проверено на текущем коде. Панель открыта (opensource) — безопасность заложена в дизайн, а не в секретность.

## Что проверено ✅
- **Секретов в публичном коде нет** — ни токенов, ни ключей, ни реальных IP флота (в отличие от «MOCK_SERVERS»
  у некоторых панелей). `.env` и локальные тест-артефакты — в `.gitignore`, в репозиторий не попадают.
- **Исходник страниц** (браузер) не выдаёт ничего секретного — только клиентский JS, который ходит в API с JWT.
  Без входа любой admin-запрос → **401**.
- **XSS** — все данные пользователей выводятся с экранированием (`esc()`); подстановка в атрибуты экранирует кавычки.
- **SQL-инъекции** — исключены (Prisma, параметризованные запросы).
- **IDOR** — admin-эндпоинты под JWT; клиентские (подписка/кабинет) — по непубличному токену устройства (24 байта),
  токен ведёт только к своему клиенту.
- **Вебхуки оплат** — обязательная проверка подписи КАЖДОГО провайдера (иначе оплата не засчитывается).
- **Двойные начисления** — атомарный переход платежа pending→paid (updateMany-guard).
- **Rate-limit** — вход 5/мин, подписка 30/мин, установщик 20/мин, весь API 120/мин.
- **Ноды** — mTLS (чужой без нашего CA не подключится) + Bearer JWT.
- **Аудит** — изменяющие действия админки логируются, секреты вычищаются.

## ⚠️ nginx — известные уязвимости (обновлять!)
Если ставишь Kand за nginx — держи его обновлённым. Актуальные проблемы:

- **CVE-2025-1974 «IngressNightmare»** (CVSS 9.8, RCE) — в **Kubernetes ingress-nginx** (не в обычном nginx!).
  Если разворачиваешь в Kubernetes — обнови ingress-nginx до **1.12.1+ / 1.11.5+** и НЕ выставляй admission-webhook наружу.
  ([Wiz](https://www.wiz.io/blog/ingress-nginx-kubernetes-vulnerabilities), [ProjectDiscovery](https://projectdiscovery.io/blog/ingressnightmare-unauth-rce-in-ingress-nginx))
- **CVE-2025-23419** (SSL session reuse) — обычный nginx; исправлено в **1.27.4+ / 1.26.3+**. Держи версию не ниже.
- **CVE-2025-53859** (mail-модуль, утечка памяти) — Kand mail-модуль nginx НЕ использует; собирай nginx без `ngx_mail`.

### Правило (заложить в деплой)
1. **Проще и безопаснее — Caddy** (авто-TLS, память-безопасный Go) как в `docs/install.md`. Рекомендуется по умолчанию.
2. Если nginx: версия **≥ 1.27.4**, только нужные модули (без mail), TLS 1.2/1.3, security-заголовки, авто-обновления пакетов.
3. В Kubernetes: ingress-nginx **≥ 1.12.1/1.11.5**, admission-webhook закрыт network-policy.
4. Панель — только по HTTPS; смени `ADMIN_PASSWORD` и `JWT_SECRET`.

## Ноды под нагрузкой
Локальный фаервол (`HARDEN_FIREWALL=1`) отбивает конн-флуд, но НЕ объёмный DDoS — для этого хостинг с анти-DDoS.
